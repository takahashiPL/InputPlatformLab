## 今回の位置づけ（T35 §5 A/D 判断同期）

- **本書は D**。今回の再開では **A を本体、D を補助**として扱う。
- 本書は **A を読むときに `axis=*` を読み違えないための補助**に限定する。
- **B/C は今回まだ触らない**。render policy 変更・実装変更・ログ文言変更には入らない。


## A 読解結論との関係

- **A の結論**は、`T17 logs = axis=mode`、`committed = axis=committed`、**両者は別軸**という読み分けである。
- 本書はその結論を支える **補助表**に留める。
- ここから **数値を揃える話・実装変更・B/C への前進**は行わない。

## 各ラベルが指す読みの束（§5 に対応）

| 接頭辞 | 何を読み分けるためのラベルか（一次情報の言い回しに対応） |
|--------|--------------------------------------------------------|
| `axis=mode` | **T17 のウィンドウ話**（モード・枠・クライアント再生成まわり）の文脈としてログを読む |
| `axis=committed` | **T14 committed**（Enter 確定の幅・高さ）の文脈として読む |
| `axis=offscreen` | **`[T34][RT]` / `[T36][RT]`** のオフスクリーン経路の文脈として読む |
| `axis=final` | **最終 HUD 帯**の文脈として読む |

**§5 の注意（そのままの趣旨）**: 数値が一致しない場面は **「軸が違う」**ことがある。
