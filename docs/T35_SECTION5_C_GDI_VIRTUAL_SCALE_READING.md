## 現在の GDI は何基準か（§2 要約）

- **Windowed**: **クライアント座標系**のまま（`Win32DebugOverlay` 既定）。
- **Borderless**: **クライアント実ピクセル**上に描画。T34 の「仮想レンダリング解像度」と**ピクセル対応は取らない**（実クライアント基準）。
- **Fullscreen**: **クライアント座標系**（全面クライアント HDC）。

## committed 仮想解像度との関係

- Borderless では D3D/D2D 側が **committed W×H** のオフスクリーンで描き、バックバッファ（全面クライアント）へ拡大合成する方針は §2.2 で固定。
- **GDI オーバーレイ**は **フルクライアント**で、**仮想 committed 座標と 1:1 にはならない**（§4 留意）。
- t34 の位置づけ表: GDI オーバーレイは **クライアント論理ピクセル**（従来どおり）。

## 「スケールするか」が意味すること / 意味しないこと

- **意味する**: §5 C どおり、将来の**別タスク**として、GDI を **committed が表す仮想レンダリング解像度**に合わせて**スケールするか**を検討する領域がある。
- **意味しない**: §2・§4 で固定された**現行の別軸**（D3D 側の committed と、クライアント基準 GDI）を、本文で**廃止すること**ではない。採否や実装の結論も出さない。
